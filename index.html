Sorting Algorithm Comparison

Project 1: Integration of Mergesort & Insertion Sort

In Mergesort, when the sizes of subarrays are small, the overhead of many recursive calls makes the algorithm inefficient. Therefore, we combine Mergesort with Insertion Sort to create a hybrid sorting algorithm for better efficiency. This involves setting a small integer S as a threshold for the size of subarrays. Once the size of a subarray in a recursive call of Mergesort is less than or equal to S, the algorithm switches to Insertion Sort, which is efficient for small-sized input.

Algorithm Comparisons

Insertion Sort

Insertion Sort is a simple sorting algorithm that builds a sorted array one item at a time. It is efficient for small data sets and can be implemented with minimal space. The time complexity is O(n^2) in the worst case.

Merge Sort

Merge Sort is a divide-and-conquer algorithm that sorts arrays by recursively splitting them into halves, sorting each half, and merging them back together. It has a time complexity of O(n log n), making it efficient for large data sets.

Hybrid Sort

The Hybrid Sort combines Merge Sort and Insertion Sort, applying Insertion Sort for small subarrays (defined by a threshold S). This approach takes advantage of the efficiency of Insertion Sort for small sizes while maintaining the O(n log n) time complexity of Merge Sort for larger sizes.
